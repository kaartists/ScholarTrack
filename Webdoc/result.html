Performance Report

Usability Analysis

Summary of our System Usability Scale (SUS) evaluation and participant feedback metrics.

90%
Positive Acceptance

55% of users agreed and 35% strongly agreed that they would use the system frequently.

85%
Ease of Use

A large majority found the interaction comfortable and completed tasks without major difficulties.

65%
Technical Dependency

Users indicated a need for technical support, suggesting a move toward "walk-up-and-use" readiness.

Key Issues Identified

  • 01 High Technical Dependency: Participants recorded a high average score (3.8/5.0) regarding the need for technical support, indicating the interface is not yet "walk-up-and-use" ready.
  • 02 Performance & Responsiveness: Qualitative feedback highlighted "delays when loading pages," which negatively impacts the perceived efficiency of the system.
  • 03 Complex Onboarding: Users felt they needed to learn many things before becoming proficient (3.7/5.0), suggesting that features are not intuitively placed.
  • 04 Lack of Data Persistence: A critical missing feature identified by users was the Auto-save function, leading to fears of data loss during long sessions.
  • 05 Interface Inconsistency: A score of 3.35/5.0 for inconsistency suggests that UI elements (buttons, forms, navigation) may behave differently across various pages.

    • Discussion of Findings

    The results reveal a "Usability Gap." Users expressed a high intent to use the system frequently (4.25/5.0) and praised its ability to make scholarship tracking "simple and organized." This indicates that the functional value of ScholarTrack is high.

    However, the SUS score was dragged down by the complexity and learning curve. The feedback suggests that the system is currently "expert-oriented"—meaning it works well once someone explains it, but it fails to guide a new user through the process independently. The demand for "mobile-friendliness" and "clearer notifications" indicates that students expect a modern, snappy experience similar to the social media apps they use daily.

    Recommendations for improving the prototype

  • 01 Self-Service Onboarding: Implement "Coach Marks" to explain features to new students.
  • 02 Optimize Performance: Reduce loading times during page transitions.
  • 03 Data Persistence: Add auto-save triggers to prevent progress loss.
  • 04 Status Visuals: Replace text status with visual progress bars.

    • Suggested Design Improvements

    Tooltip Integration

    Add small "i" icons next to scholarship requirements that show detailed descriptions on hover.

    Status Progress Bars

    Instead of simple text status, use a visual progress bar (e.g., Submitted > Under Review > Approved) to give users immediate visual feedback.

    Responsive Grid System

    Re-adjust the Figma prototype to include a mobile-view layout, ensuring that all buttons are "thumb-friendly" and text is legible on smaller screens.

    Centralized Notification Center

    Replace static updates with a dedicated notification bell icon that alerts users to status changes or new scholarship deadlines.
